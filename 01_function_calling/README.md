# 第 1 课：Function Calling

让模型「调用你的函数」的最小闭环。

```bash
python 01_function_calling/demo.py
```

## 盯住三件事（面试就考这个）

1. **模型从不自己执行函数**，它只「返回意图」(`tool_calls`)，执行权在你的代码里。
2. **工具结果要回填给模型** (`role:"tool"`)，它才能接着推理 —— 这就是 ReAct 里的 **Observation**。
3. **`for` 循环 + 最大步数** = Agent 的本质 + 防失控（跟防无限 render 一样）。

## 数据流

```
用户问题
  → 模型读每个工具的 description，决定调哪个、怎么填参数  (③ 模型只吐意图)
  → 你的代码执行真正的函数                                (你 = 执行权)
  → 结果以 role:"tool" 回填进 messages                    (ReAct 的 Observation)
  → 模型拿到结果，生成自然语言答案                          (第二次出场)
```

## mock vs 真模型

- **没配 `.env`**：用规则模拟的「假模型」，零成本，照样看懂流程。
- **配了 `.env`**：真 Azure 模型**自己**读 description、自己决定调不调、怎么填参数——
  这才是面试官想听的「我真跑过，看过模型自主决策」。

试着把问题改得刁钻一点（比如「我想知道深圳冷不冷顺便帮我找个充电宝」），
看真模型会不会**一次吐两个 tool_call** —— 这就是多工具编排的雏形。
