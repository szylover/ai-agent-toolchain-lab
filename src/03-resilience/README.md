# 第 3 课：失败处理（Resilience）

工具调用不可靠时，runtime 怎么扛住。**这一课不需要模型**——失败处理全在「执行工具」那一层。
这正是前端最熟的活：超时、重试、兜底。

```bash
npm run resilience
```

## 三板斧（面试高频）

1. **超时控制 timeout** —— 工具卡住不能拖死整个 agent（`withTimeout` 用 `Promise.race` 思路实现）。
2. **指数退避重试 retry + backoff** —— 抖动型故障（网络/限流/5xx）重试往往就好；退避 + 抖动避免重试风暴。
3. **降级兜底 fallback** —— 重试用尽仍失败，返回约定的安全默认值，**绝不让流程崩**。

## demo 演了两条路径

- **场景 A**：天气服务第 1 次超时、第 2 次报 503、第 3 次成功 → **重试救场**。
- **场景 B**：股价服务彻底挂 → 重试到头 → **降级返回 `{price:-1}`**（上层可识别为「不可用」并友好提示）。

## 面试一句话

> 工具调用要包一层 runtime 的弹性策略：超时防止被拖死、指数退避重试应对抖动型故障、
> 重试用尽则降级兜底保证 agent 不崩。这层和模型无关，纯属 runtime 工程化。
