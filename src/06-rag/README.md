# 第 6 课：RAG（检索增强生成）

> 跑：`npm run rag`

## 一句话
**RAG = 先去知识库【检索】相关片段，拼进 prompt，再让模型【生成】。**
模型本身不动（还是那个纯函数），变的只是「你喂给它的上下文」。

## 为什么需要 RAG（面试常考动机）
模型的知识有三个硬伤，RAG 正好对症：

| 硬伤 | 例子 | RAG 怎么解 |
|---|---|---|
| **不知道私有数据** | 公司内网文档、用户的订单 | 把私有库检索后喂进去 |
| **知识会过期** | 今天的价格 / 库存 | 检索实时数据源 |
| **记不住长尾细节** | 某条 SLA 的具体数字 | 用原文兜底，避免瞎编 |
| **会幻觉** | 一本正经编造 | 要求「只依据资料作答 + 标引用」 |

本课用一份**虚构**的 `ZeroStore` 内部文档（`corpus.ts`），模型预训练时绝不可能见过，
所以能清楚看到：**裸问 → 瞎编一通；RAG → 精确命中 48.8 TB / 60 天 / 7 天恢复，还带 [引用]。**

## 工具链 5 步（`rag.ts` 逐段对应）
```
① 切块 chunk     长文档 → 小片段（检索的最小单位）
② 向量化 embed   片段 → 向量（语义指纹）
③ 入库 index     向量存进向量库（本课就是内存数组）
④ 检索 retrieve  问题也向量化 → 算相似度 → 取 top-k     ← 全程无 LLM！
⑤ 增强+生成      top-k 拼进 prompt → 模型据此作答 + 标引用 ← 唯一用 LLM 的一步
```
关键认知：**①~④ 一个 token 都不烧**，纯工程检索；LLM 只在第 ⑤ 步出现。
所谓「RAG 工具链」，90% 的工程量在检索侧，不在模型侧。

## 本课的玩具实现 vs 生产实现
| 环节 | 本课（零依赖、能跑通） | 生产 |
|---|---|---|
| 向量化 | 中文 字+二字 词袋（`tokenize`） | embedding 模型（text-embedding-3、bge、m3e…） |
| 相似度 | 余弦相似度（手写） | 同样是余弦/点积，但在向量库里算 |
| 向量库 | 内存数组 + 全量扫描 | 专用向量库（pgvector、Milvus、Pinecone、FAISS） |
| 检索 | 纯语义 top-k | 混合检索（向量 + BM25 关键词）+ 重排 rerank |

> 用词袋是为了让你**不配 embedding 部署也能完整跑通整条链路**、看清形状。
> 换成真 embedding，代码结构一行不用变——这正是「面向接口编程」的好处。

## 进阶概念（面试加分点）
- **Chunking 切块策略**：太大→检索不准、塞爆上下文；太小→语义破碎。常用「按语义/标题切 + 重叠 overlap」。
- **Hybrid Search 混合检索**：向量擅长语义、BM25 擅长精确关键词（型号、错误码），二者融合召回更稳。
- **Re-ranking 重排**：先粗召回 top-50，再用 cross-encoder 精排出 top-5，兼顾速度与精度。
- **Citation 引用**：让模型在每个事实后标 `[编号]`，可溯源、可核查，是企业级 RAG 的硬要求（见 `buildMessages`）。
- **失败模式**：① 检索没召回到 → 模型「资料中未提及」（本课已要求，避免幻觉）；② 召回到但不相关 → 噪声干扰；③ 上下文太长 → "lost in the middle"。

## 面试问答速记
**Q：RAG 和微调（fine-tune）怎么选？**
A：RAG 适合「知识频繁变、要可溯源、私有数据」；微调适合「改变模型的风格/格式/固有能力」。
知识更新用 RAG（改库即可，不用重训）；行为定制用微调。常见是两者结合。

**Q：上下文窗口越来越大，RAG 是不是没用了？**
A：不是。① 全量塞进去贵且慢，还有 "lost in the middle"；② 私有/实时数据仍需检索拉取；
③ 可溯源、可控权限（只检索用户有权看的片段）。长上下文是补充不是替代。

**Q：RAG 答错了怎么排查？**
A：先分层定位——**是检索没召回到（retrieval 问题），还是召回到了但模型没用好（generation 问题）**？
前者调切块/embedding/混合检索/重排；后者调 prompt（强调"只依据资料"、加引用约束）。

**Q：怎么评估 RAG 质量？**
A：拆成两段指标：检索侧用 recall@k / MRR；生成侧用 faithfulness（忠实于资料、不幻觉）、answer relevance。
工具如 RAGAS。

## 和工具链其它课的关系
- RAG 本质也可以包成一个 **tool**（`searchKnowledgeBase(query)`），交给 agent 在 function calling 里调用——
  那样就从「固定 RAG 流水线」升级成「agent 自己决定要不要检索、检索几次」。本课先掌握固定流水线。
